## 📌 Описание проекта
Прототип игры в жанре auto-survivor в стиле Vampire Survivors.  
Игрок управляет персонажем, который автоматически атакует врагов, получает опыт, повышает уровень и улучшает способности.

Проект выполнен как тестовое задание на позицию Junior Unity Developer в компанию Dankolab с обязательным активным использованием AI-инструментов.

---

## ⚙️ Функциональность

### Основной геймплей
- 🎮 Управление персонажем — WASD / стрелки (8 направлений) + левый стик на геймпаде
- 🤖 Автоматические атаки по ближайшему врагу
- ⭐ Система опыта — враги дропают опыт (ExperienceOrb)
- 📈 Система уровней — при апе открывается меню улучшений
- 🛑 Пауза при открытии различных меню

---

### Умения

| Умение | Описание | Прокачиваемые параметры |
|--------|----------|--------------------------|
| Projectile | Выстрелы в ближайших врагов | Урон, скорость, скорострельность |
| Aura | Постоянный урон и замедление вокруг игрока | Урон, радиус, замедление |
| Melee | Урон врагам в радиусе ближнего боя | Урон, радиус, интервал атаки, отбрасывание |
| Magnet | Притяжение сфер опыта | Радиус притяжения, скорость притяжения |
| Health | Увеличение здоровья и регенерация | Максимальное HP, реген |
| Speed | Увеличение скорости передвижения | Скорость движения |

---

### Враги

| Тип | Поведение | Особенности |
|------|------------|--------------|
| Normal | Медленно движется к игроку | Базовый враг |
| Fast | Быстрое преследование | Меньше здоровья, выше скорость |
| Shooter | Стреляет | Имеет дальнобойную атаку |

## 📦 Баланс через JSON

Все параметры загружаются из JSON-файлов:

- Статы игрока  
- Параметры умений  
- Параметры врагов  
- Настройки волн и прогрессии  

Основной конфиг:  
`Assets/Resources/GameConfig.json`

## 🗂️ Архитектура проекта

Assets/
│
├── Resources/
│   ├── GameConfig.json          # Главный файл баланса и конфигурации
│   └── Prefabs/
│       ├── Enemies/             # Префабы врагов (Normal, Fast, Shooter)
│       ├── Player/              # Префаб игрока
│       └── Projectiles/         # Префабы снарядов
│
├── Scenes/
│   └── MainScene.unity          # Основная сцена игры
│
├── Scripts/
│   ├── Combat/                  # Логика боевых систем
│   │   ├── Projectile.cs        # Логика снарядов игрока
│   │   ├── EnemyProjectile.cs   # Логика снарядов врагов
│   │   ├── AuraRangeEffect.cs   # Аура урона и замедления
│   │   └── MeleeRangeDetector.cs# Ближний бой и отбрасывание
│   │
│   ├── Data/                    # Классы данных и конфигурации
│   │   ├── GameConfig.cs        # Корневой класс JSON-конфига
│   │   ├── PlayerData.cs        # Статы игрока
│   │   ├── EnemyData.cs         # Статы врагов
│   │   ├── UpgradeData.cs       # Данные улучшений
│   │   ├── WaveData.cs          # Параметры волн
│   │   └── ExperienceData.cs    # Параметры прогрессии и опыта
│   │
│   ├── Enemy/                   # Поведение врагов
│   │   ├── EnemyController.cs   # Базовый класс врага
│   │   ├── EnemyFast.cs         # Быстрый враг
│   │   └── EnemyShooter.cs      # Стреляющий враг
│   │
│   ├── Player/                  # Поведение игрока
│   │   └── PlayerController.cs  # Движение, стрельба, взаимодействие
│   │
│   ├── ItemUI/                  # UI элементов прокачки
│   │   └── UpgradeItemUI.cs     # Один элемент апгрейда в меню
│   │
│   ├── Managers/                # Центральные менеджеры игры
│   │   ├── GameManager.cs       # Управление состоянием игры, UI, волнами, паузой
│   │   └── GameConfigManager.cs # Загрузка и управление JSON-конфигурацией
│   │
│   └── Systems/                 # Вспомогательные игровые системы
│       ├── ExperienceOrb.cs     # Сферы опыта и их притяжение
│       ├── BulletPool.cs        # Object Pool для снарядов
│       └── WaveSpawner.cs       # Система спавна врагов по волнам
│
└── UI/
    ├── HUD/                     # Основной интерфейс (HP, XP, волна)
    ├── LevelUpPanel/            # Панель выбора улучшений
    ├── PauseMenu/               # Меню паузы
    └── GameOverPanel/           # Экран окончания игры

## ✍️ Что дорабатывалось вручную

- Добавление Object Pool для снарядов
- Использование Input System
- Интеграция Cinemachine
- Переработка Data-классов
- Проектирование JSON-структуры
- Замена публичных полей на private + Properties
- Настройка объектов и UI-компонентов
- Доработка проверок на null
- Оптимизация часто вызываемых методов
- Создание вспомогательных методов для читаемости
- Полная настройка UI и отображения статов
- Добавление метода SetGamePaused() и вставка в нужные места
- Создание UpgradeItemUI, т.к. из префаба не получилось получить объекты напрямую через .transform.Find()
- Исправление DamageFlash: сохранение исходного цвета объекта в Awake() для корректного возврата
